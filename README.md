# Дерево выражений (Expression Tree)

Реализация структуры данных **Дерево выражений** на языке Python. Программа принимает математическое выражение в **постфиксной форме** (обратная польская запись), строит на его основе бинарное дерево, рекурсивно вычисляет итоговый результат, а также умеет переводить выражение в **префиксную форму**.

---

## 🚀 Основные возможности

В проекте успешно реализованы:
1. **Обязательная часть**:
   * Функция `build_expression_tree(tokens)` — синтаксический разбор постфиксной записи и построение бинарного дерева.
   * Функция `evaluate(root)` — рекурсивный обход дерева для вычисления арифметического результата.
2. **Вариативная часть (Пункт 3)**:
   * Функция `to_prefix(root)` — обход дерева в порядке *Pre-order* для генерации префиксной (польской) записи.
3. **Дополнительно**:
   * Функция `print_tree(root)` — псевдографическая визуализация структуры получившегося дерева прямо в консоли.

---

## 🧠 Как это работает? Суть алгоритма

### 1. Построение дерева (`build_expression_tree`)
Алгоритм использует структуру данных **Стек** (Stack) для обработки списка токенов слева направо:
* Если встречается **число**, для него создается узел-лист, который помещается в стек.
* Если встречается **оператор** (`+`, `-`, `*`, `/`):
  1. Из стека извлекаются два верхних узла.
  2. Важно: первый извлеченный узел становится **правым** потомком, а второй — **левым** (это критично для некоммутативных операций, таких как деление и вычитание).
  3. Создается новый внутренний узел с этим оператором, а извлеченные узлы связываются с ним.
  4. Новый узел-родитель отправляется обратно в стек.

В конце выполнения в стеке остаётся ровно один узел — это **корень дерева**.

### 2. Вычисление значения (`evaluate`)
Вычисление происходит с помощью классического рекурсивного обхода:
* Базовый случай: если узел является листом (числом), мы просто возвращаем его значение.
* Если узел — оператор, функция рекурсивно считает значения левого и правого поддеревьев, а затем применяет оператор корня к полученным результатам.

### 3. Префиксная форма (`to_prefix`)
Для получения префиксной формы дерево обходится в порядке **Корень ➔ Левый ➔ Правый**:
* Сначала записывается сам оператор, а затем результаты обхода его левой и правой веток. 
* Для выражения `(2 + 3) + (4 * 5)` префиксный вид будет: `+ + 2 3 * 4 5`.
